# 교육세션 태블릿 컨텐츠 및 데이터 관리 설계 (Education Session Design)

## 1. 개요
매장(라스브러리) 방문 회원의 태블릿으로부터 학습 데이터를 서버로 동기화하고, 대용량(약 4GB)의 차수별 신규 컨텐츠(APK)를 USB-C 케이블을 통해 고속으로 전송 및 설치하는 시스템 설계입니다.

## 2. 아키텍처 (Architecture)

### 2.1 로컬 브리지 방식 (Local Bridge)
브라우저의 대용량 전송 불안정성을 해결하기 위해 매장 PC에 상주하는 소형 **'로컬 에이전트(ADB Bridge)'**를 사용합니다.

- **Admin UI (Browser)**: 관리자가 조작하는 웹 화면.
- **Local Agent (Native)**: PC에 설치된 실행 파일. 브라우저와 통신하며 실제 USB 명령 수행.
- **ADB (Android Debug Bridge)**: 태블릿과 직접 통신하여 파일 전송 및 APK 설치 담당.

## 3. 주요 기술 요건

### 3.1 하드웨어 및 인프라
- **케이블**: USB-C 3.1 Gen2 이상 권장 (4GB 전송 시 수십 초 내 완료).
- **매장 PC**: SSD 탑재 권장 (대용량 파일 버퍼링 및 ADB 처리 속도 확보).
- **태블릿**: Android 기반 (USB 디버깅 모드 활성화 필요).

### 3.2 소프트웨어 및 프로토콜
- **전송 엔진**: `adb push` 또는 `adb install -r` 명령어를 로컬에서 직접 호출.
- **데이터 동기화**:
    1. 태블릿 내 특정 경로의 데이터 파일(`JSON` or `DB`)을 PC로 읽어옴.
    2. 관리 시스템 API(`itbookmanager`)를 통해 회원 정보에 기록.
    3. 기록 성공 확인 후 태블릿 내 원본 데이터 삭제(`adb shell rm`).
- **컨텐츠 업데이트**: 컨텐츠 서버로부터 예정된 차수의 APK를 다운로드하여 태블릿에 전송.

### 3.3 관리용 태블릿(Android) 사용 시 시나리오
지점 PC 대신 관리자용 태블릿을 활용하여 학습용 태블릿을 관리할 경우의 추가 요건입니다.

- **USB 호스트(Host) 지원**: 관리용 태블릿이 USB 호스트 역할을 수행해야 하므로 **USB-C OTG(On-The-Go)** 기능이 필수적입니다.
- **네이티브 관리 앱 필수**: 안드로이드 브라우저(Chrome 등)는 외부 USB 장치에 대해 4GB 파일을 쓰고 설치하는 권한(WebUSB)이 제한적입니다. 따라서 **전용 네이티브 앱(APK)**을 통해 USB 포트를 제어해야 합니다.
- **전원 공급 문제**: 두 기기가 직접 연결되면 한 기기가 다른 기기를 충전하게 되어 배터리 소모가 극심해질 수 있습니다. **PD 충전이 지원되는 USB 허브** 사용이 권장됩니다.
- **하이브리드 방식 권장**: 매장 내 PC(허브)를 두고, 관리용 태블릿에서는 웹 화면을 통해 PC에 "전송 시작" 명령만 내리는 방식을 통해 기동성과 안정성을 동시에 확보할 수 있습니다.

## 4. 상세 워크플로우 (Workflow)

1. **태블릿 연결**: 회원이 방문하여 지점 PC의 USB-C 포트에 태블릿 연결.
2. **인식 및 검사**: 로컬 에이전트가 태블릿 일련번호 인식 및 현재 설치된 APK 버전/학습 데이터 유무 확인.
3. **학습 데이터 수집**: 기존 학습 기록을 추출하여 서버 API로 전송.
4. **회원 정보 업데이트**: 서버에서 학습 데이터 수신을 확인하고 회원 진도율 업데이트.
5. **신규 컨텐츠 다운로드**: 다음 차수의 4GB APK를 컨텐츠 서버에서 로컬 PC 캐시로 로드.
6. **초고속 전송 및 설치**: USB-C 대역폭을 최대한 활용하여 태블릿에 신규 APK 설치.
7. **완료 및 분리**: 전송 완료 알림 후 태블릿 분리 및 교실 입장.

## 5. 보안 및 저작권 보호 (Security & Copyright)
본 시스템은 4GB 대용량 컨텐츠의 저작권 유출을 방지하기 위해 단계별 보안책을 적용합니다.

### 5.1 로컬 데이터 격리 및 유출 방지
- **임시 저장 및 즉시 삭제**: 컨텐츠 서버에서 내려받은 APK는 로컬 PC의 암호화된 임시 디렉토리에 저장되며, 태블릿 전송 완료 즉시 복구가 불가능한 방식으로 영구 삭제(Shredding)됩니다.
- **파일명 난독화 및 권한 제어**: 내려받는 파일은 무작위 문자열로 이름을 변경하며, 관리자 권한 없이는 탐색기에서 접근할 수 없도록 OS 레벨의 폴더 권한을 격리합니다.
- **메모리 기반 전송 (RAM Disk)**: 보안 수준을 극대화할 경우, 로컬 디스크가 아닌 시스템 메모리(RAM) 공간을 가상 드라이브로 할당하여 컨텐츠를 적재합니다. 시스템 종료 시 데이터는 물리적으로 소멸됩니다.

### 5.2 암호화 및 무단 복제 방지
- **On-the-fly 복호화**: 서버에서 암호화된 상태로 전송받은 뒤, 로컬 에이전트가 메모리 상에서만 복호화하여 태블릿으로 스트리밍합니다.
- **중복 전송 방지**: 승인된 회원 단말기(기기 식별자 일치)에 대해서만 단 1회 전송을 허용하는 서버 측 토큰 시스템을 연동합니다.

## 6. 고려 사항
- **오프라인 대응**: 매장 인터넷이 불안정할 경우를 대비하여 컨텐츠는 미리 PC에 로컬 캐싱 처리하되, 위 보안 규정을 준수함.
- **파일 무결성**: 4GB 대용량 전송이므로 전송 후 파일 체크섬(Hash) 검사 필수.
- **보안**: 태블릿과의 최초 연결 시 'USB 디버깅 허용' 팝업 승인 절차 가이드 필요.
