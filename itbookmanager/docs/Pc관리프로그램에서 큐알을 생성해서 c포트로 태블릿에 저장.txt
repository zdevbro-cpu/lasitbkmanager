Pc관리프로그램에서 큐알을 생성해서 c포트로 태블릿에 apk로 전송 
PC → device_id 생성 → 태블릿에 전달 → APK가 QR 생성
디바이스에 c타입포트 연결 시 시리얼정보를 확인, 매칭된 태블릿 QR 전송

👉 APK는 “하나만 사용”
👉 태블릿마다 “값만 다르게 주입”

구현방법
방법 1 (가장 추천) : ADB + 파일 주입 방식

✔️ 전체 흐름
PC 프로그램에서 device_id 생성
텍스트 파일 생성
C타입(USB) 연결
태블릿에 파일 전송
APK가 파일 읽어서 QR 생성

✔️ PC에서 파일 생성
device_id=TAB-0001

파일명:
device.txt

✔️ PC → 태블릿 전송 (ADB)
adb push device.txt /sdcard/device.txt

✔️ APK에서 읽기
File file = new File("/sdcard/device.txt");
BufferedReader br = new BufferedReader(new FileReader(file));
String line = br.readLine();
String deviceId = line.split("=")[1];

👉 이 값을 QR로 생성

✔️ 장점
APK 1개만 유지
태블릿별 식별 가능
C포트 연결만으로 세팅 완료


👉 아래 조합 추천드립니다:

✔️ APK → QR 생성 기능 포함
✔️ device_id → 파일(/sdcard/device.txt)로 주입
✔️ PC → ADB 자동화

👉 이유:

안정성 최고
디버깅 쉬움
기기 변경 대응 가능
device_id 파일은 삭제되지 않도록 관리
APK 실행 시 없으면:
→ “미등록 기기” 표시
QR + 텍스트 같이 표시 (현장 대응)

🔥 최종 결론

👉 “QR을 APK에 넣는 방식”이 아니라
👉 “device_id를 주입하고 APK가 QR 생성” 방식
