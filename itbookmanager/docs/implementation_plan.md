# 컨텐츠 관리 및 라이프사이클 설계 (개발 계획서)

학습 시스템을 위한 컨텐츠 서버로 구글 드라이브(Google Drive)의 적합성을 평가하고, 보다 견고한 컨텐츠 관리 및 학생 학습 상태 업데이트 아키텍처를 제안합니다.

## 분석: 구글 드라이브를 컨텐츠 서버로 사용 시 (프로토타입)

### 장점
- **친숙한 인터페이스:** 비기술직 직원도 UI를 통해 파일 업로드 및 관리가 용이함.
- **비용 무료:** 소규모 프로토타입의 경우 초기 비용이 발생하지 않음.
- **접근 제어:** 공유 폴더 기능을 통한 간단한 권한 관리 가능.

### 단점 및 기술적 문제 (중요)
> [!WARNING]
> 구글 드라이브는 본래 CDN이나 대용량 컨텐츠 서버용으로 설계되지 않았습니다.
- **속도 제한(Rate Limiting):** API 쿼터가 존재하여 여러 태블릿에서 동시 다운로드 시 403 오류가 발생할 수 있음.
- **직접 다운로드의 복잡성:** 대용량 파일은 URL에 `confirm=t` 등의 파라미터가 필요하며, 이를 위해 중계 서버(Proxy) 로직이 복잡해질 수 있음.
- **CORS 미지원:** 브라우저/앱에서 구글 드라이브로의 직접적인 AJAX/fetch 요청이 차단됨.
- **지연 시간(Latency):** 전용 스토리지 서비스에 비해 파일 목록 조회 및 다운로드 속도가 느림.

---

## 스토리지 서비스 비교: Supabase vs Firebase/GCP

| 비교 항목 | Supabase Storage | Firebase Storage (GCP) |
| :--- | :--- | :--- |
| **활용성 (Usability)** | **매우 높음.** 기존 Supabase DB(Auth)와 완벽 연동. SQL 권한 제어(RLS) 가능. | **높음.** Firebase Auth/Firestore와 연동 시 유용. 모바일 앱 특화 기능 많음. |
| **유연성 (Flexibility)** | **중상.** 기본적인 웹/앱 요구사항에 충분. S3 호환 가능. | **최상.** GCP 클라우드 에코시스템(Vision API, 영상 인코딩 등) 연계 용이. |
| **가격 (Pricing)** | **가성비 우수.** 프로(Pro) 요금제($25)에 100GB 포함. 송출(Egress) 비용이 예측 가능함. | **사용량 기반.** 무료 구간(5GB) 이후 사용량만큼 과제. 대량 트래픽 시 비용 급증 가능성. |
| **결론** | **현재 프로젝트(`las-mgmt`)에 최적.** 백엔드를 분산하지 않고 통합 관리 가능. | 분석/AI 등 구글 클라우드 특정 기능이 필수인 경우에 적합. |

## 추천하는 선택: Supabase Storage

현재 `las-mgmt`에서 이미 Supabase를 사용하고 있으므로, **Supabase Storage**가 가장 유리합니다.

1. **통합 관리:** 별도의 인증(Auth) 로직을 구글 서비스와 분리하여 구현할 필요가 없습니다.
2. **개발 생산성:** 현재 지점/도서 정보가 DB에 있으므로, "이 지점에서만 다운로드 가능" 같은 제어를 DB 쿼리처럼 쉽게 짤 수 있습니다.
    - 예: `auth.uid() in (select manager_id from branches where branch_id = ...)`
3. **학습 데이터 연동:** 학습자의 `cha_su` 상태와 실제 컨텐츠 파일 간의 정합성을 유기적으로 관리하기에 가장 좋습니다.

## 인증 시스템(Auth) 이관 검토: Firebase에서 Supabase로

현재 Firebase Auth를 사용 중이라면, Supabase로의 이관은 **아키텍처 통합성** 측면에서 매우 강력한 이점이 있습니다.

### 이관 시 장점
1. **강력한 보안 통합 (RLS):** Supabase Auth의 유저 ID(`uid`)가 데이터베이스와 스토리지 권한 제어에 즉시 반영됩니다. 별도의 중간 API 서버 검증 로직 없이도 안전하게 태블릿에서 파일을 다운로드할 수 있습니다.
2. **개발 단순화:** 하나의 SDK(`supabase-js`)만 관리하면 됩니다. Firebase와 Supabase 두 종류의 인증 토큰을 관리해야 하는 복잡함이 사라집니다.
3. **데이터 관계의 명확성:** 유저 테이블(`auth.users`)을 프로젝트의 프로필 테이블과 SQL Foreign Key로 직접 연결할 수 있어 데이터 품질 관리가 쉽습니다.

### 이관 시 고려사항 (Hurdles)
1. **기존 사용자 데이터 이전:** Firebase 유저 데이터(이메일, 비밀번호 해시 등)를 추출하여 Supabase로 마이그레이션하는 작업이 필요합니다 (Supabase는 Firebase Scrypt 해시 형식을 지원함).
2. **로그인 기능 재구현:** 소셜 로그인(Google 로그인 등)의 클라이언트 ID/Secret 연동 설정을 Supabase 대시보드에서 다시 해야 합니다.

## 최종 제안: 단계적 통합

당장 모든 사용자를 이관하기 부담스럽다면 다음과 같은 단계를 제안합니다:
- **1단계 (임시):** Firebase Auth를 유지하되, 서버에서 Firebase 토큰을 검증한 후 Supabase 전용 임시 토큰(Service Role)을 발급하여 스토리지 접근.
- **2단계 (권장):** 신규 프로젝트(태블릿 관리/LMS)부터는 Supabase Auth로 시작하고, 점진적으로 전체 유저를 통합.

## 제안하는 학습 라이프사이클 흐름

1. **정보 조회:**
   - 태블릿 앱이 `student_id`와 함께 API를 호출.
   - API는 `learning_status` 테이블에서 현재 학습 중인 차수(cha_su) 정보를 확인.
2. **상태 업데이트 및 정리:**
   - API에 "다운로드 시작" 이벤트를 기록.
   - 앱 내에서 *이전 차수*의 로컬 데이터 및 학습 상태를 삭제.
3. **다음 차수 다운로드:**
   - API가 다음 차수의 ZIP 컨텐츠에 대한 "서명된 URL(Signed URL)"(예: 30분 유효)을 생성하여 전달.
   - 앱이 컨텐츠를 다운로드하고 압축을 해제.
4. **완료 처리:**
   - 앱이 "다운로드 성공" 신호를 API에 전송.
   - API는 `current_cha_su`를 다음 값으로 업데이트하고 다음 차수를 활성화.

## Supabase 요금제 및 Egress(송출 트래픽) 분석

### Egress란 무엇인가요?
**Egress(송출 트래픽)**는 Supabase 서버에서 외부(태블릿, 사용자 브라우저 등)로 나가는 데이터의 양을 의미합니다.
- **예:** 태블릿이 Supabase Storage에서 100MB 크기의 컨텐츠 파일을 다운로드하면, 100MB의 Egress가 발생합니다.
- **참고:** 파일을 업로드하는 것(Ingress)은 보통 무료이며, 데이터베이스 쿼리 결과로 나가는 텍스트 데이터도 Egress에 포함되지만 양이 매우 적습니다.

### 프로 플랜($25/월) 적합성 검토
현재 `las-mgmt` 프로젝트와 신규 컨텐츠 서버 기능을 고려할 때, **프로 플랜 사용을 적극 권장**합니다.

1.  **제공량:** 프로 플랜은 월 **50GB의 무료 Egress**를 포함합니다.
2.  **예상 사용량 계산 (예시):**
    - 컨텐츠 파일당 200MB 기준, 월 250회 다운로드 발생 시 약 50GB 소모.
    - 50GB를 초과하더라도 서비스가 차단되지 않고, GB당 약 $0.09(약 120원)의 저렴한 비용으로 자동 결제되어 운영이 안정적입니다.
3.  **스토리지 혜택:** 프로 플랜은 기본 8GB의 스토리지를 제공하며($25 기본료에 포함), 이후 사용량만큼 추가 비용이 발생하므로 프로토타입을 넘어 실서비스 운영 시 파일 용량 제한 걱정이 없습니다.
4.  **백업 및 성능:** `las-mgmt`와 같은 관리 시스템에서 중요한 데이터베이스 백업(Daily Backup) 기능이 활성화되어 데이터 안전성이 높아집니다.
